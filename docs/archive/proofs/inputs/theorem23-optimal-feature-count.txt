# Theorem 23: Optimal Feature Count Under Bias-Variance Tradeoff

We extend Theorem 19 (ApproximationGainVsEstimationCost) to the setting where features are ranked by decreasing marginal importance and one must choose how many to include.

## Definitions

Let n > 0 be the number of training samples. Let d_max be the total number of available features. For k = 1, ..., d_max, define:

- B(k): the approximation bias when using the top k features. B is non-increasing in k (more features reduce bias).
- V(k): the estimation variance when using k features. V is non-decreasing in k (more features increase variance).

Assume B(k) = B_0 * exp(-alpha * k) for some B_0 > 0, alpha > 0 (exponential decay of marginal importance, consistent with the observed permutation importance curve).

Assume V(k) = lambda * k / n for some lambda > 0 (variance grows linearly with feature count and inversely with sample size, standard statistical learning theory).

Define the expected loss (risk):

  L(k) = B(k)^2 + V(k) = B_0^2 * exp(-2*alpha*k) + lambda*k/n.

Prove the following claims.

1. L(k) has a unique minimizer k* over the positive reals, given by

     k* = (1 / (2*alpha)) * log(2 * alpha * n * B_0^2 / lambda).

   Prove this by setting dL/dk = 0 and verifying the second derivative is positive.

2. k* increases logarithmically in n (more data justifies more features) and decreases with lambda (higher estimation cost per feature penalizes complexity).

3. For k < k*, adding one more feature reduces loss (bias reduction dominates). For k > k*, adding one more feature increases loss (variance increase dominates).

4. Numerical claim. With the following empirical estimates:
   - n = 250000 (training samples)
   - B_0 = 0.30 (baseline bias, estimated from Sortino gap between 0-feature and full model)
   - alpha = 0.10 (importance decay rate: top feature drop=0.30, 10th feature drop=0.18, roughly exp(-0.10*k))
   - lambda = 0.005 (estimation cost per feature, estimated from v5 vs v10 gap scaled by feature ratio)

   Compute k* and verify that it falls in the range [10, 20]. This would confirm that 9 features is slightly below optimal and 31 is above optimal, consistent with v10 (9 feat, Sortino=0.161) nearly matching v5 (31 feat, Sortino=0.177).

5. Define the relative efficiency RE(k) = L(k*) / L(k). Prove that RE(k) is symmetric around k* on a log scale, meaning using too few features is equally costly as using too many by the same multiplicative factor.
