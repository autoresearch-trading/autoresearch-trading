# Documentation Guide

This directory groups the project knowledge base according to the phases outlined in `vision/MASTER_IDEA.md`.

- `vision/` – long-term strategy, success criteria, and high-level guardrails.
- `architecture/` – system design, project structure, and implementation plans.
- `operations/` – deployment, maintenance, and day-to-day runbooks.
- `reports/` – historical test reports, paper trading notes, and validation summaries.
- `archive/` – legacy status snapshots and CI notes kept for reference.

Each subdirectory owns its own README with additional context when necessary. Start with `vision/MASTER_IDEA.md` to understand the product direction, then drill into the other sections as you execute the roadmap.
