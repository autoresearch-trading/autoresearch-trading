# Theorem 38: Timestamp Alignment via Nearest-Before Matching

## Definitions

Consider two time series:
- **Trade batches** with timestamps T = (t_1, t_2, ..., t_N) where t_1 < t_2 < ... < t_N (strictly increasing)
- **Orderbook snapshots** with timestamps S = (s_1, s_2, ..., s_M) where s_1 < s_2 < ... < s_M (strictly increasing)

The **nearest-before alignment** maps each trade batch timestamp t_i to the latest orderbook snapshot at or before t_i:

  align(t_i) = s_{j*}  where j* = max{j : s_j <= t_i}

If no s_j <= t_i exists, the batch has no aligned snapshot (missing data).

In code, this is implemented via a linear scan:

```
ob_idx = 0
for i in range(N):
    while ob_idx < M-1 and S[ob_idx+1] <= T[i]:
        ob_idx += 1
    if ob_idx < M and S[ob_idx] <= T[i]:
        result[i] = data[ob_idx]
```

An equivalent implementation uses searchsorted:

```
indices = np.searchsorted(S, T, side="right") - 1
```

## Claims

### Part A: Correctness of Nearest-Before

1. **Unique alignment.** For each t_i with at least one s_j <= t_i, the aligned index j* = max{j : s_j <= t_i} is unique (since S is strictly increasing). Prove this.

2. **No future leakage.** For all aligned pairs, s_{j*} <= t_i. The model never uses orderbook data from AFTER the trade batch. Prove this from the definition.

3. **Freshest available.** Among all valid alignments, j* gives the most recent snapshot: there is no j' > j* with s_{j'} <= t_i. Prove this (j* is the maximum by definition).

4. **Staleness bound.** The staleness of the alignment is t_i - s_{j*} >= 0. If orderbook snapshots arrive at intervals of at most Delta (max_{j} (s_{j+1} - s_j) <= Delta), then staleness <= Delta for all aligned batches (except possibly the first). Prove this.

### Part B: Equivalence of Implementations

5. **Linear scan equals searchsorted.** The linear scan implementation and `searchsorted(S, T, side="right") - 1` produce the same alignment. Prove this:

   - `searchsorted(S, t_i, side="right")` returns the index k such that S[k-1] <= t_i < S[k] (or M if t_i >= all of S).
   - Therefore k-1 = max{j : s_j <= t_i} = j*.
   - The linear scan maintains ob_idx as the running j* and advances it forward, which gives the same result because T is sorted.

6. **Linear scan is O(N + M).** The linear scan visits each element of S at most once (ob_idx only moves forward). Combined with the loop over N batches, total work is O(N + M). Prove this.

   Searchsorted is O(N log M) (binary search for each of N queries). The linear scan is faster when N and M are similar in magnitude.

### Part C: Edge Cases

7. **No snapshot before first batch.** If s_1 > t_1, the first batch has no aligned snapshot. All feature values derived from the orderbook for this batch should be 0 (default). Prove that the alignment function correctly returns "missing" for this case.

8. **Multiple batches between snapshots.** If t_i < t_{i+1} < ... < t_{i+k} all fall between s_j and s_{j+1}, they all align to s_j (the same snapshot). This means they all get the same orderbook features. Prove that this is consistent with the "nearest-before" semantics and does not introduce lookahead bias.

9. **Snapshot after all batches.** If s_M > t_N, the final snapshot is never used. This is correct (no future data used). Prove this.

### Part D: Cache Consistency

10. **Cache key determinism.** The cache key is computed as MD5(f"{symbol}_{start}_{end}_{trade_batch}_{feature_version}"). Prove that:
    - Two calls with identical (symbol, start, end, trade_batch, feature_version) produce the same key (deterministic)
    - Changing any one parameter produces a different key (with high probability, since MD5 collisions are negligible for short strings)
    - Bumping feature_version invalidates all caches for all symbols and splits (since the version string changes in every key)
