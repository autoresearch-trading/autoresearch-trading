# Theorem 45: Execution Latency as a Cost Component

## Definitions

Let L > 0 be the execution latency (time between signal generation and order fill) in seconds.
Let delta_t > 0 be the duration of one model step in seconds.
Let sigma_step > 0 be the per-step return standard deviation.

The price uncertainty during latency is:
  sigma_latency = sigma_step * sqrt(L / delta_t)

assuming returns are i.i.d. across sub-step intervals.

Let S > 0 be the bid-ask spread (in return units).
Let impact > 0 be the market impact buffer (in return units).

The total execution cost per trade is:
  C_total = S/2 + impact + E[|price_slippage_from_latency|]

## Claims

1. **Latency cost as fraction of spread.** The expected absolute price change during latency L is:
     E[|Delta_P / P|] = sigma_step * sqrt(L / delta_t) * sqrt(2/pi)

   (using the expected absolute value of a normal random variable with std = sigma_latency).

   Prove that E[|Z|] = sqrt(2/pi) for Z ~ N(0,1), and therefore:
     E[|Delta_P / P|] = sqrt(2/pi) * sigma_step * sqrt(L / delta_t)

2. **Latency is subsumed by spread model.** If the spread S already reflects the market maker's adverse selection cost, which includes compensation for latency-induced price movement, then:
     S/2 >= E[|Delta_P / P| during latency]

   holds when the market maker's quote lifetime exceeds the execution latency. In this case, the latency cost is NOT additive to the spread — it is already priced in.

3. **Numerical verification.** With our parameters:
   - L = 0.4 seconds (400ms Solana block time)
   - delta_t = 1.0 seconds (approximate step duration for BTC at 100 trades/batch)
   - sigma_step = 0.001024 (empirical per-step std)

   Compute:
     latency_cost = sqrt(2/pi) * 0.001024 * sqrt(0.4 / 1.0)
                  = 0.7979 * 0.001024 * 0.6325

   Verify that latency_cost < 0.00052 (less than 0.52 bps).
   Compare to our spread: median BTC spread = 0.1 bps, half_spread = 0.05 bps.
   Verify that latency_cost > half_spread for BTC (latency exceeds the narrow BTC spread).
   But compare to impact_buffer = 3 bps: latency_cost / impact_buffer < 0.2.

4. **Signal decay during latency.** If the model's predictive signal has autocorrelation rho_signal at lag L/delta_t, the fraction of signal surviving after latency is:
     signal_retained = rho_signal(L / delta_t)

   From Theorem 47, our signal has autocorrelation < noise_threshold at lag >= 1. Since L/delta_t = 0.4 < 1 step, the signal at execution time is:
     signal_retained = rho_signal(0.4) > rho_signal(1) ≈ 0

   By continuity and the fact that rho_signal(0) = 1, the signal retained is high (close to 1) for sub-step latency. Prove that for any stationary process with rho(0) = 1 and rho continuous:
     rho(x) → 1 as x → 0

5. **Conclusion.** Execution latency of 400ms on Solana DEX perps does NOT need to be modeled as a separate cost because:
   a) The latency cost (< 0.52 bps) is small relative to the impact buffer (3 bps)
   b) The spread model (half_spread + impact_buffer) already accounts for execution uncertainty
   c) Signal decay during sub-step latency is negligible (signal retained ≈ 1)
   d) MEV (sandwich attacks) does not affect perpetual futures on DEXs (targets AMM swaps instead)
